# Any — упрощённая реализация std::any на C++

## Описание проекта

В рамках данного проекта реализован упрощённый аналог класса std::any, появившегося в стандарте C++17.

Класс позволяет хранить объекты произвольных типов в одном контейнере и получать их обратно с проверкой типа во время выполнения.

Основная цель проекта — изучение механизма Type Erasure (стирание типов), динамического полиморфизма, шаблонов и работы с умными указателями.


---

## Постановка задачи

Необходимо реализовать класс Any, поддерживающий хранение значений произвольного типа.

Пример использования:

Any a(42);

a = std::string("Hello");

a = true;

Для извлечения значения используется функция:

auto str = AnyCast<std::string>(a);

Если сохранённый тип не совпадает с запрашиваемым, должно быть выброшено исключение:

BadAnyCast

Также необходимо реализовать:
- конструктор по умолчанию;
- конструктор копирования;
- оператор присваивания;
- поддержку перемещения;
- метод Reset();
- метод Swap();
- метод HasValue();
- функцию MakeAny().

---

Используемая техника — Type Erasure

Реализация основана на механизме стирания типов.

Вместо хранения объекта неизвестного типа напрямую используется иерархия классов:

Any
 │
 └── unique_ptr<IHolder>
               │
      ┌────────┴─────────┐
      │                  │
AnyHolder<int>    AnyHolder<string>
      │                  │
     42              "Hello"

Базовый интерфейс

class IHolder
{
public:
    virtual std::unique_ptr<IHolder> Clone() const = 0;
    virtual ~IHolder() = default;
};

Шаблонная обёртка

template<class T>
class AnyHolder : public IHolder
{
    T value_;
};

Каждый конкретный тип автоматически получает собственную реализацию контейнера.

## Архитектура проекта

BadAnyCast

Пользовательское исключение для обработки ошибок приведения типов.

class BadAnyCast : public std::bad_cast


---

IHolder

Абстрактный базовый класс.

Содержит виртуальный метод:

Clone()

который позволяет копировать объект без знания его реального типа.


---

AnyHolder<T>

Шаблонный наследник IHolder.
- Хранит объект типа T.

Поддерживает:
- хранение значения;
- копирование через Clone();
- получение значения через Value().



---

Any

Основной контейнер.

Внутри хранит:
- std::unique_ptr<IHolder> holder_;

Поддерживает:
- хранение объектов произвольного типа;
- копирование;
- перемещение;
- очистку содержимого;
- обмен содержимым;
- проверку наличия значения.

---

Реализованные методы
- Конструктор по умолчанию
- Создаёт пустой объект.

Any a;


---

Конструктор от произвольного типа

Any a(42);


---

Конструктор копирования

Any a(42);
Any b(a);

Копирование выполняется через виртуальный метод Clone().


---

Оператор присваивания

Поддерживается как присваивание другого объекта Any:

a = b;

так и произвольного типа:

a = 10;
a = std::string("abc");


---

HasValue()

Проверка наличия значения.

if (a.HasValue())
{
    ...
}


---

Reset()

Удаление сохранённого объекта.

a.Reset();

После вызова объект становится пустым.


---

Swap()

Обмен содержимым двух объектов.

a.Swap(b);


---

AnyCast

Получение значения нужного типа.

int x = AnyCast<int>(a);

Проверка выполняется через: dynamic_cast

Если типы совпадают — возвращается сохранённое значение.

Если типы различаются: throw BadAnyCast();


---

MakeAny

Вспомогательная функция создания объекта.

Пример:

auto a = MakeAny<std::string>(5, 'x');

Эквивалентно:

Any a(std::string(5, 'x'));


---

### Примеры использования

Хранение различных типов

Any a(42);

a = std::string("Hello");

a = true;


---

### Извлечение значения

Any a(100);

int value = AnyCast<int>(a);


---

### Ошибка приведения типа

Any a(std::string("Hello"));

int value = AnyCast<int>(a);

Результат:

BadAnyCast error

## Итог

Проект демонстрирует один из классических способов реализации контейнера для хранения объектов произвольного типа в C++ посредством техники Type Erasure.

Реализация повторяет базовую функциональность std::any и позволяет на практике изучить взаимодействие шаблонов, динамического полиморфизма и управления памятью через умные указатели.

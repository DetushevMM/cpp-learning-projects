## Задача 1. `CStringSentinel`: итератор-сентинел для C-style строки

### Условие
Стандартные алгоритмы C++ обычно принимают диапазон в виде пары итераторов (`begin`, `end`).  
Для C-style строки это часто означает предварительный расчет длины, чтобы получить конец диапазона, что добавляет дополнительный проход.

Требуется:
- реализовать класс `CStringSentinel`, который завершает итерацию при достижении `'\0'`;
- с помощью `std::ranges::all_of` проверить, что все символы строки являются буквами;
- вывести `YES`, если условие выполняется, иначе `NO`.

Ограничение на длину входной строки: до `10^6` символов.

### Идея решения
- Создается sentinel-тип `CStringSentinel` с перегруженным `operator==` для сравнения с `const char*`.
- Сравнение возвращает `true`, когда текущий символ равен `'\0'`, то есть это конец диапазона.
- Алгоритм `std::ranges::all_of` запускается как `all_of(str.c_str(), CStringSentinel{}, IsLetter)`.
- Предикат `IsLetter` проверяет принадлежность символа диапазонам `A-Z` и `a-z`.

### Почему это хорошее решение
- Один проход по данным без отдельного вычисления длины C-style строки.
- Использование `std::ranges` делает код декларативным и легко читаемым.
- Подход масштабируется для длинных строк и других sentinel-ориентированных сценариев.

### Сложность
- Время: `O(n)`
- Память: `O(1)`

## Задача 2. Свободные клетки: фильтрация стартов окна через `ranges`

### Условие
Дана строка из символов `'.'` (свободная клетка) и `'#'` (занятая клетка), а также число `k`.  
Нужно вывести все позиции `i`, для которых отрезок длины `k`, начинающийся в `i`, полностью свободен (состоит только из `'.'`).

Требование задачи: использовать `std::views::iota` и `std::views::filter`.

Ограничения:
- длина строки `n` до `10^7`;
- `1 <= k <= n`;
- индексация с нуля.

### Идея решения
- Используется техника скользящего окна длины `k`.
- Переменная `blocked` хранит количество символов `'#'` в текущем окне.
- Сначала `blocked` вычисляется для первого окна `[0, k)`.
- Далее формируется ленивый диапазон индексов `0..m-1` (где `m = n - k + 1`) через `views::iota`.
- В `views::filter` для каждого `i`:
  - проверяется условие `blocked == 0` (окно полностью свободно);
  - окно сдвигается на 1 позицию вправо (для следующего `i`), обновляя `blocked` за `O(1)`.
- Все подходящие позиции печатаются в одну строку через пробел.

### Почему это хорошее решение
- Требования по `ranges` соблюдены: используется связка `iota + filter`.
- Нет повторного пересчета каждого окна длины `k`, только инкрементальные обновления.
- Подходит для больших входов (до `10^7`) благодаря линейному проходу и `O(1)` дополнительной памяти.

### Сложность
- Время: `O(n)`
- Память: `O(1)` (не считая входной строки)

## Задача 3. CSV-парсер: сумма чисел через `views::split` и `views::filter`

### Условие
Дана одна CSV-строка (поля разделены запятыми), где поля могут быть:
- целыми числами;
- строками;
- пустыми значениями.

Нужно вычислить сумму всех числовых полей.

Требование задачи: использовать `std::views::split` и `std::views::filter`.

Ограничения:
- до `10^6` полей;
- числовые значения по модулю не больше `100`.

### Идея решения
- Входная строка разбивается на поля через `views::split(',')`.
- Для каждого поля собирается временная строка и проверяется, является ли оно корректным целым числом:
  - разрешен ведущий `-`;
  - после `-` должны идти только цифры;
  - пустая строка не считается числом.
- Через `views::filter` остаются только числовые поля.
- Каждое числовое поле конвертируется через `std::stoi` и добавляется к сумме.

### Почему это хорошее решение
- Логика парсинга выражена декларативно и читаемо через `ranges`.
- Пустые и строковые поля автоматически исключаются на этапе фильтрации.
- Решение масштабируется на большие CSV-строки без сложных структур данных.

### Сложность
- Время: `O(L)`, где `L` — длина входной строки
- Память: `O(1)` дополнительной памяти (кроме временного поля при проверке/конвертации)

## Задача 4. Фильтруем слова: потоковая обработка через `std::ranges`

### Условие
Даны:
- число `n` — количество слов;
- число `k` — минимальная допустимая длина слова;
- далее `n` слов из латинских букв.

Нужно вывести только слова длины не меньше `k`, предварительно переведя их в нижний регистр, сохранив исходный порядок.

Ограничения задачи также запрещают:
- циклы со счетчиками/итераторами для обработки коллекций результата;
- промежуточные контейнеры;
- рекурсию.

### Идея решения
- Слова читаются по одному и обрабатываются сразу (streaming-подход).
- Если длина слова меньше `k`, оно пропускается.
- Для подходящих слов применяется `std::ranges::transform` по символам:
  - символы `A-Z` переводятся в `a-z`;
  - остальные остаются без изменений.
- Преобразованное слово сразу выводится в поток.

### Почему это хорошее решение
- Нет промежуточного хранения: память остается минимальной.
- Каждое слово обрабатывается ровно один раз, что хорошо для больших `n`.
- Используются инструменты `std::ranges`, как требует условие.

### Сложность
- Время: `O(S)`, где `S` — суммарная длина всех слов
- Память: `O(1)` дополнительной памяти (кроме текущего слова)

## Задача 5. Поступашки-2: top-k через `partition` и сортировку с проекцией

### Условие
Дан список абитуриентов в формате:
`<Имя> <Математика> <Русский> <Физика>`.

Нужно вывести `top-k` абитуриентов:
- учитываются только те, у кого каждый предмет не меньше `40`;
- сортировка по убыванию суммы трех баллов;
- при равной сумме — по имени в алфавитном порядке.

Требования задачи:
- использовать сортировку `ranges` с параметром проекции;
- не использовать компараторы;
- не использовать циклы со счетчиками/итераторами (кроме чтения входа);
- не вычислять и не хранить сумму заранее.

### Идея решения
- Все абитуриенты читаются в один вектор структур (`name`, `math`, `rus`, `phys`).
- `std::ranges::partition` выполняет in-place разбиение:
  - слева остаются прошедшие порог (`>= 40` по каждому предмету),
  - справа — остальные.
- Сортировка (`std::ranges::sort`) применяется только к левой части (успешным).
- Для выполнения требований без компаратора используется проекция в `std::tuple`:
  - первый ключ: `-SumScore()` (дает порядок по убыванию суммы),
  - второй ключ: `name` (лексикографический tie-break).
- На вывод берутся первые `min(k, count_successful)` имен через `std::views::take`.

### Почему это хорошее решение
- `partition` + сортировка только релевантного префикса обычно эффективнее, чем сортировать весь массив и потом фильтровать.
- Все операции выполняются in-place, без дополнительных контейнеров для отфильтрованного списка.
- Проекция в `tuple` элегантно заменяет компаратор и соблюдает ограничения условия.

### Сложность
- Время: `O(n) + O(m log m)`, где `m` — число прошедших абитуриентов (`m <= n`)
- Память: `O(1)` дополнительной памяти (кроме входного массива)

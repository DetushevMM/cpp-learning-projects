# Описание задачи

На складе хранятся контейнеры с товарами N различных видов. Все контейнеры составлены в N стопок. В каждой стопке могут находиться контейнеры с товарами любых видов (стопка может быть изначально пустой).

Автопогрузчик может взять верхний контейнер из любой стопки и поставить его сверху в любую стопку. 
Необходимо расставить все контейнеры с товаром первого вида в первую стопку, второго вида – во вторую стопку и т.д.

Программа должна вывести последовательность действий автопогрузчика или сообщение о том, что задача решения не имеет.

Формат ввода
В первой строке входных данных записано одно натуральное число N, не превосходящее 500. 
В следующих N строках описаны стопки контейнеров: сначала записано число ki – количество контейнеров в стопке.
Затем ki чисел – виды товара в контейнерах в данной стопке, снизу вверх. 
В каждой стопке вначале не более 500 контейнеров (в процессе переноса контейнеров это ограничение может быть нарушено).

Формат вывода
Программа должна вывести описание действий автопогрузчика: 
- для каждого действия напечатать два числа – из какой стопки брать контейнер и в какую стопку класть.
Обратите внимание, что минимизировать количество операций автопогрузчика не требуется. 
Если задача не имеет решения, необходимо вывести одно число 0.

Пример
Ввод:
3 
4 1 2 3  2
0
0                        
Вывод:
1 2
1 3
1 2
Объяснение примера: 
Изначально в первой стопке лежат четыре контейнера – снизу контейнер с товаром первого вида, над ним – с товаром второго вида, над ним третьего, и сверху еще один контейнер с товаром второго вида. 
Вторая и третья стопки – пусты.

📦 Containers Sorting (C++)

📌 Описание проекта

Реализация алгоритма сортировки контейнеров на складе.

На складе:

Хранятся контейнеры N различных видов

Контейнеры расположены в N стопках

В каждой стопке могут находиться контейнеры любых видов

Автопогрузчик может:

взять верхний контейнер из любой стопки

положить его на вершину любой стопки



🎯 Цель

Распределить контейнеры так, чтобы:

В стопке №1 — контейнеры вида 1

В стопке №2 — контейнеры вида 2

...

В стопке №N — контейнеры вида N


Программа выводит последовательность действий автопогрузчика.

Минимизация количества операций не требуется.

Если задача не имеет решения — выводится 0.


---

🏗 Архитектура решения

Структура данных

Используется стек на основе односвязного списка:

struct Node {
    int type;
    Node* next;
};

struct Stack {
    Node* top;
};

Реализованные операции

Push() — добавить контейнер в стопку

Pop() — удалить верхний контейнер

Top() — получить верхний контейнер

Empty() — проверка на пустоту

Clear() — освобождение всей памяти


Все операции выполняются за O(1).


---

🧠 Алгоритм

Общий случай (N > 2)

1. Читаем входные данные.


2. Контейнеры, уже находящиеся в правильной стопке (внизу), игнорируются.


3. Для перемещений используется стратегия:

Если целевая стопка пуста → перенос напрямую.

Если занята → используется последняя стопка как буфер.


4. После обработки всех стопок разбирается буферная стопка.



---

Особый случай (N = 2)

Для двух стопок проверяется корректность возможного распределения.
Если конфигурация нарушает порядок — выводится 0.

Время O(M), где M — общее количество контейнеров
Память O(M)

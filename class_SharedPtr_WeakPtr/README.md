# SharedPtr / WeakPtr — учебная реализация

Упрощённый аналог `std::shared_ptr` и `std::weak_ptr` на C++ с разделяемым владением и подсчётом сильных и слабых ссылок. Решение оформлено как **один заголовочный файл** `shared_ptr.hpp` (header-only).

---

## Описание задачи

Умные указатели реализуют идиому **RAII**: ресурс привязывается к времени жизни объекта, освобождение памяти не требует ручных вызовов `delete`.

**`std::shared_ptr`** (C++11) даёт **разделяемое владение**: несколько указателей владеют одним объектом в куче; объект удаляется, когда не остаётся «сильных» владельцев. Для разрыва циклических ссылок в стандарте есть **`std::weak_ptr`**: он не продлевает жизнь объекта, но позволяет при необходимости получить `shared_ptr` через `lock()`, если объект ещё жив.

На практике `shared_ptr` дороже `unique_ptr` из‑за счётчиков и синхронизации в многопоточных реализациях стандарта; учебная версия фокусируется на **логике владения и счётчиков**, а не на оптимизации до уровня стандартной библиотеки.

---

## Условия (основное задание)

Реализовать шаблон **`SharedPtr<T>`** — упрощённый аналог `std::shared_ptr`.

Идея: хранить указатель на объект и **счётчик сильных ссылок** в динамической памяти (чтобы все копии `SharedPtr` ссылались на **одни и те же** счётчики и один объект). При создании копии счётчик увеличивается, при уничтожении — уменьшается; при обнулении сильных ссылок **объект удаляется**.

### Требуемый интерфейс `SharedPtr`

| Элемент | Назначение |
|--------|------------|
| Конструктор по умолчанию | Нулевой указатель |
| Конструктор от `T*` | Владение переданным объектом (новый блок счётчиков / логика «нового» владения) |
| Копирование и копирующее присваивание | Новая сильная ссылка на тот же объект |
| Перемещение и перемещающее присваивание | Передача владения без лишнего инкремента/декремента у источника |
| `Reset(T* ptr = nullptr)` | Смена управляемого указателя |
| `Swap(SharedPtr&)` | Обмен состояниями |
| `Get()` | Сырой указатель `T*` |
| `UseCount()` | Число сильных ссылок |
| `operator*` / `operator->` | Доступ к объекту |
| `explicit operator bool()` | Проверка на ненулевой указатель |

Справочник: [std::shared_ptr на cppreference](https://en.cppreference.com/w/cpp/memory/shared_ptr).

Формат сдачи: **один файл** `shared_ptr.hpp`.

## Дополнительное задание 1 — `WeakPtr`

Реализовать **`WeakPtr<T>`** по аналогии с `std::weak_ptr`.

Рекомендуемая модель данных: структура **`Counter`** (или эквивалент) в куче с полями **`strong_count`** и **`weak_count`**, на которую указывают и `SharedPtr`, и `WeakPtr`.

### Правила освобождения памяти

1. Если **сильных и слабых** ссылок не осталось — удаляются **и объект, и счётчик(и)**.
2. Если **сильных** стало 0, но **слабые** ещё есть — удаляется **только объект**, блок счётчиков **живёт**, пока живы `WeakPtr`.

### Интерфейс `WeakPtr`

Конструкторы (по умолчанию, копирование, перемещение), присваивания, конструктор от `SharedPtr`, `Swap`, `Reset`, `UseCount` (число **сильных**), `Expired()`, `Lock()` → пустой `SharedPtr`, если объект уже уничтожен.

В **`SharedPtr`** добавить конструктор от **`WeakPtr`**: как `Lock()`, но если объект «протух» — исключение **`BadWeakPtr`**.

Для проверяющей системы в заголовке должна быть директива **`#define WEAK_PTR_IMPLEMENTED`**.

Справочник: [std::weak_ptr на cppreference](https://en.cppreference.com/w/cpp/memory/weak_ptr).

## Решение (кратко)

В репозитории/архиве с решением лежит **`shared_ptr.hpp`**, в котором объявлены:

- **`BadWeakPtr`** — наследник `std::runtime_error`, выбрасывается при конструировании `SharedPtr` от «протухшего» `WeakPtr`.
- **`SharedPtr<T>`** — владеет `T*` и разделяет с другими `SharedPtr`/`WeakPtr` два счётчика в куче: число сильных и число слабых ссылок (по смыслу соответствует вынесенному **Counter**, реализованному как два `size_t*`).
- **`WeakPtr<T>`** — не владеет объектом; увеличивает слабый счётчик; в деструкторе корректно освобождает контрольный блок, когда суммарных ссылок не осталось.
- Приватный конструктор **`SharedPtr(T*, size_t*, size_t*)`** — для **`Lock()`**: создаёт новую сильную ссылку на уже существующий объект без повторного выделения объекта.
- Деструктор **`SharedPtr`** реализует правила: при `strong == 0` удаляется объект; если при этом `strong + weak == 0`, удаляются и счётчики.

В конце файла объявлено **`#define WEAK_PTR_IMPLEMENTED`**.

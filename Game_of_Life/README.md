Conway's Game of Life (C++)

📖 Описание проекта

Данный проект представляет собой реализацию клеточного автомата «Игра Жизнь» (Conway's Game of Life) на языке C++.

Игра Жизнь была предложена британским математиком Джоном Конвеем в 1970 году и является одной из самых известных моделей клеточных автоматов.

Каждая клетка игрового поля может находиться в одном из двух состояний:

- "*" — живая клетка;
- "-" — мёртвая клетка.

Состояние каждой клетки в следующем поколении определяется количеством её живых соседей.

⚙️ Правила игры

Для каждой клетки игрового поля вычисляется количество живых соседей (максимум 8).

Если клетка живая ("*")

- остаётся жить, если имеет 2 или 3 живых соседа;
- умирает, если имеет меньше 2 соседей (одиночество);
- умирает, если имеет больше 3 соседей (перенаселение).

Если клетка мёртвая ("-")

- оживает, если имеет ровно 3 живых соседа.

---

🗂 Структура данных

Для хранения игрового поля используется структура:

struct Field {
    int rows;
    int cols;
    char** cells;
};

Где:

- "rows" — количество строк;
- "cols" — количество столбцов;
- "cells" — динамический двумерный массив клеток.

---

📁 Формат входного файла

Исходная конфигурация хранится в текстовом файле.

Пример:

10 10
1 2
2 3
3 1
3 2
3 3

Первая строка содержит размеры поля:

rows cols

Все последующие строки содержат координаты живых клеток:

row col

В приведённом примере будут созданы живые клетки:

(1,2)
(2,3)
(3,1)
(3,2)
(3,3)

Все остальные клетки считаются мёртвыми.

🏗 Архитектура программы

Программа состоит из набора функций:

Функция| Назначение
"CreatWorld2D()"| Создание динамического поля
"FillWorld2D()"| Заполнение поля мёртвыми клетками
"DeleteWorld2D()"| Освобождение памяти
"PrintField2D()"| Вывод поля на экран
"LoadFieldFromFile()"| Загрузка начальной конфигурации
"CountNeighbours()"| Подсчёт живых соседей клетки
"GetNextCellStatus()"| Определение состояния клетки следующего поколения
"CreateNextGeneration()"| Создание нового поколения
"CompareFields()"| Сравнение двух поколений
"CountLiveCells()"| Подсчёт живых клеток
"ChangeField()"| Обмен поколений

---

🚀 Алгоритм работы программы

1. Загружается начальная конфигурация из файла.
2. Создаётся игровое поле.
3. Создаётся поле следующего поколения.
4. Выводится текущее состояние Вселенной.
5. Вычисляется следующее поколение.
6. Проверяются условия завершения.
7. При необходимости процесс повторяется.

---

🛑 Условия завершения

Симуляция завершается если:

1. Все клетки погибли

Количество живых клеток равно нулю.

2. Конфигурация стабилизировалась

Новое поколение полностью совпадает с предыдущим.
